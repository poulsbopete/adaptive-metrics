Observability Cases · lab workflow

What the governance Case documents each run

The workflow Adaptive metrics — Retail Banking · Metric governance snapshot opens a Case whose title starts with [Governance] Adaptive Metrics — what we changed (Streams). The description lists real Streams API actions, not vague savings rhetoric.

Logs — fork + retention

POST /api/streams/logs.otel.banking/_fork for child streams such as logs.otel.banking.document-vault, logs.otel.banking.quote-engine, and logs.otel.banking.policy-manager, then PUT …/_ingest with data_retention: 14d on each child (cheaper than indefinite parent retention).

Metrics — retention only

metrics-governance.noisy.otel-default → 7d (lab OTLP governance shipper) · metrics-generic.otel-default → 14d · metrics-hostmetricsreceiver.otel-default → 14d

Modeled savings vs. real bills

Shorter retention reduces stored data going forward; it is not a guaranteed percentage off your Serverless invoice. The Case’s follow-up comment includes ES|QL FinOps proxies (e.g. estimated_monthly_observe_bill_usd, pct_savings_of_estimated_observe_bill) for prioritization—not Finance-approved forecasts. Tune days in metric-governance-retail-banking-starter.yaml or override in Observability → Streams.
